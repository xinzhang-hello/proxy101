What is proxy101

When we are behind a proxy application, we might encounter some wired problems, the top 1 is: Connection Refused.

Proxy101 aims to help you understand and debug application proxy status. Which connections connected through proxy, which application connected through proxy and which configuration they used?  For example, we use intellij ideas to develop. When a connection is refused, I want to know why the connection refused? Figure out paths like: check server status, check which configuration makes the connection connect through proxy and how to configure properly?

I will teach you how to use a different application on Windows/MacOS using proxy, you don’t want to read it all just following my best practice.

Best Practices For Windows

System scripts

Set system proxy properly Or you can set through UI which I will skip

  1. Set your proxyServer if you’re not 1080
  2. Let’s see you are in a company named bilibili, your company’s internal domain is *.bilibili.co and for sure you don’t want the traffic to bilibili.co through your proxy server. You can add it bypass list.

# Set the proxy server address and port

$proxyServer = "http://127.0.0.1:1080"

# Set the bypass list

$bypassList = "*.bilibili.co;<local>"

# Enable the proxy and set the bypass list

Set-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings' -Name ProxyServer -Value $proxyServer

Set-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings' -Name ProxyEnable -Value 1

Set-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings' -Name ProxyOverride -Value $bypassList

# Notify the system of the proxy change

$signature = @'

[DllImport("wininet.dll", SetLastError = true, CharSet=CharSet.Auto)]

public static extern bool InternetSetOption(IntPtr hInternet, int dwOption, IntPtr lpBuffer, int dwBufferLength);

'@

$type = Add-Type -MemberDefinition $signature -Name WinINet -Namespace pinvoke -PassThru

$INTERNET_OPTION_SETTINGS_CHANGED = 39

$INTERNET_OPTION_REFRESH = 37

$type::InternetSetOption(0, $INTERNET_OPTION_SETTINGS_CHANGED, 0, 0) | Out-Null

$type::InternetSetOption(0, $INTERNET_OPTION_REFRESH, 0, 0) | Out-Null

Write-Host "Proxy has been configured and enabled with the specified bypass list."

Get-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Internet Settings' | Select-Object ProxyServer, ProxyEnable, ProxyOverride

Using Powershell

# Define variables for proxy settings

$proxyHost = 127.0.0.1"

$proxyPort = "1080"

$proxyUsername = ""

$proxyPassword = ""

$nonProxyHosts = "localhost|127.0.0.1"

# Define the path to the Maven settings.xml file (modify path if necessary)

$mavenSettingsPath = "$env:USERPROFILE\.m2\settings.xml"

# Check if settings.xml exists, if not, create it with the default structure

if (-Not (Test-Path $mavenSettingsPath)) {

    # Create directory if it doesn't exist

    if (-Not (Test-Path "$env:USERPROFILE\.m2")) {

        New-Item -Path "$env:USERPROFILE\.m2" -ItemType Directory

    }

   

    # Add a basic structure for settings.xml

    @"

<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"

          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0

                              http://maven.apache.org/xsd/settings-1.0.0.xsd">

</settings>

"@ | Out-File -FilePath $mavenSettingsPath -Encoding utf8

}

# Load the settings.xml file

[xml]$xml = Get-Content $mavenSettingsPath

# Check if a proxies section exists, create it if necessary

if (-Not $xml.settings.proxies) {

    $proxiesNode = $xml.CreateElement("proxies")

    $xml.settings.AppendChild($proxiesNode) | Out-Null

}

# Create a new proxy entry

$proxy = $xml.CreateElement("proxy")

$active = $xml.CreateElement("active")

$active.InnerText = "true"

$proxy.AppendChild($active) | Out-Null

$protocol = $xml.CreateElement("protocol")

$protocol.InnerText = "http"

$proxy.AppendChild($protocol) | Out-Null

# Rename variable to avoid conflict with PowerShell's built-in $host

$proxyHostElement = $xml.CreateElement("host")

$proxyHostElement.InnerText = $proxyHost

$proxy.AppendChild($proxyHostElement) | Out-Null

$port = $xml.CreateElement("port")

$port.InnerText = $proxyPort

$proxy.AppendChild($port) | Out-Null

# If username and password are provided, add authentication details

if ($proxyUsername -and $proxyPassword) {

    $username = $xml.CreateElement("username")

    $username.InnerText = $proxyUsername

    $proxy.AppendChild($username) | Out-Null

    $password = $xml.CreateElement("password")

    $password.InnerText = $proxyPassword

    $proxy.AppendChild($password) | Out-Null

}

# Add nonProxyHosts if provided

if ($nonProxyHosts) {

    $nonProxyHostsElement = $xml.CreateElement("nonProxyHosts")

    $nonProxyHostsElement.InnerText = $nonProxyHosts

    $proxy.AppendChild($nonProxyHostsElement) | Out-Null

}

# Get the proxies node and append the new proxy

$proxiesNode = $xml.settings.proxies

$proxiesNode.AppendChild($proxy) | Out-Null

# Save the updated settings.xml file

$xml.Save($mavenSettingsPath)

Write-Host "Proxy settings updated in $mavenSettingsPath"
